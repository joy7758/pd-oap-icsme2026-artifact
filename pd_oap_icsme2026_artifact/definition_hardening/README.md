# v2.0 Definition Hardening

This package defines the conceptual layer needed to answer the TOSEM concern that several key concepts were not clearly defined.

Definition hardening comes before fixture expansion and evaluation expansion. It sets the terms that later artifact growth, mutation evaluation, and manuscript rewriting must use consistently.

## Required Concepts

- public-data operation
- operation-accountability statement
- structural inspectability
- reviewability
- semantic failure surface
- controlled fixture benchmark

## Definition Requirements

Each definition must include:

- concise definition
- scope
- what it is not
- example
- non-example
- artifact fields
- validation consequence
- software engineering role
- manuscript-use sentence

The goal is to make the v2.0 manuscript read as a software engineering research contribution, not as a loose artifact report.
