# PD-OAP v2.0 Software Engineering Framing

This package addresses the editorial criticism that the connection to software engineering was weak and not convincingly established.

PD-OAP v2.0 is framed as "a software-engineering method for specifying, validating, and evaluating operation-accountability artifacts for public-data workflows".

## Components

- definitions
- controlled fixture benchmark
- semantic checker
- mutation evaluation
- RQ-driven evaluation dashboard

The package does not modify v1.x artifacts. It also does not modify existing v2.0 fixtures, checker outputs, mutation outputs, or dashboard reports.
